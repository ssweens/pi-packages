## Decisions Made

- **Hard-coded constants live in the orchestrator file** (`BOOTSTRAP_SOURCE`, `BOOTSTRAP_MARKETPLACE_NAME`). Not user-overridable in V1; documented in the file header so a future "configurable target" feature has an obvious starting point.
- **Selective error swallow**: only `MarketplaceDuplicateNameError` is caught from `addMarketplace`. Any other failure (network, manifest, etc.) propagates so the user surface mirrors the underlying orchestrator failures.
- **Handler bypasses `parseCommandArgs`**: the schema-driven validator does not reject extra positionals on an empty `positional: []` schema, so the bootstrap shim parses with `parseArgs` directly and asserts `positional.length === 0`. This was anticipated by the plan and matches the pattern verbatim.
- **`--scope user` is also rejected**: belt-and-suspenders so the user contract ("bootstrap always targets user scope; do not pass `--scope`") cannot be muddled by a quietly-accepted explicit `--scope user`.
