<objective>
Make agent `model:` field emission opt-in. By default, the generated Pi agent
`.md` file produced by the agents bridge MUST omit the `model:` field entirely
(letting Pi pick its own default). A new `--map-model` boolean flag on
`/claude:plugin install` and `/claude:plugin update` restores the existing
AG-7 model-mapping behavior (sonnet/opus/haiku/inherit/unknown).

Purpose: Per CONTEXT.md decisions, mapping is changing from always-on to
opt-in. Pi has gained its own model-default behavior and the V1 always-on
mapping is now a footgun for users who do not want the imported Claude agent
to override Pi's default. The change must be:

  - threaded cleanly through edge -> orchestrator -> bridge -> convertAgent;
  - applied to BOTH install and update (per CONTEXT.md "--map-model scope");
  - NOT applied to the marketplace autoupdate cascade (`updateSinglePlugin`),
    which has a fixed signature and is not user-driven;
  - covered by tests at the bridge layer (field omission) and edge layer
    (flag plumbing);
  - reflected in PRD §5.2.1, §5.2.3, and AG-7 detail.

Output:
  - Source: convert.ts, stage.ts, types.ts, two orchestrators, two edge
    handlers, completions provider updated.
  - Tests: convert.test.ts gets new omit-by-default and explicit-map-model
    cases; pre-existing AG-7 model-mapping tests get `mapModel: true` added
    so they continue to assert the mapping path. Edge install/update shim
    tests gain a `--map-model` acceptance case.
  - Docs: PRD updated (AG-7 detail line 417; §5.2.1 line 245; §5.2.3 line
    280) to describe the opt-in default.
</objective>
