# Quick Task 260516-08j: modify agent mapping logic to omit model unless --map-model option is passed to plugin install command - Context

**Gathered:** 2026-05-16
**Status:** Ready for planning

<domain>
## Task Boundary

Modify the agent artifact mapping logic so that the mapped Pi agent file omits the
`model` field by default. A new `--map-model` flag on `plugin install` and `plugin
update` opts back in to model mapping.

</domain>

<decisions>
## Implementation Decisions

### --map-model scope
- The `--map-model` flag applies to both `plugin install` and `plugin update`
- `update` re-installs agents, so it must also honour the flag

### Agent file output when model omitted
- When `--map-model` is NOT passed, the generated agent `.md` file must have NO
  `model` field at all (not null, not empty string -- simply absent)
- Pi picks its own default model in this case

### Artifact scope
- PRD (`docs/prd/pi-claude-marketplace-prd.md`) must be updated to reflect the new
  default and the `--map-model` opt-in
- VALIDATION.md must be updated if any CMP requirement coverage changes
- Implementation (agent-builder / mapper code) and tests must be updated

### Claude's Discretion
- Exact PRD section(s) to amend -- locate by reading current PRD text on agent mapping
- Whether any new CMP requirement is warranted or existing ones are amended
- Test strategy for verifying field omission vs field presence

</decisions>
