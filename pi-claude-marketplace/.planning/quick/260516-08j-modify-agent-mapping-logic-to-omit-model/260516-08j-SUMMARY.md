# Quick Task 260516-08j: Modify agent mapping logic to omit model

## One-liner

AG-7 model mapping is now opt-in via `--map-model` on `/claude:plugin install` and `/claude:plugin update`; the default omits `model:` from generated Pi agent files so Pi picks its own default. Cascade autoupdate always omits `model:`.

- **bridges/agents/stage.ts** -- Destructures `mapModel` from `StageAgentsInput` and forwards `mapModel: mapModel ?? false` into every `convertAgent` call inside step 3 of `prepareStagePluginAgents`.

## Cascade path: intentionally left unchanged

`updateSinglePlugin` (the `PluginUpdateFn` cascade entrypoint invoked by the marketplace autoupdate cascade) and its inner `runThreePhaseUpdate({ ..., cascade: true })` call were left untouched. The `PluginUpdateFn` signature has no `--map-model`, and cascade-driven re-installs MUST always use the omit-by-default behavior so they don't surprise users by overriding Pi's default model. `args.mapModel` is undefined on that path, which resolves to `false` at the bridge call site via `args.mapModel ?? false` in `prepareUpdateHandles`. This is documented inline on the `mapModel?: boolean` field of `ThreePhaseArgs`.

## Tests touched outside the planned files

Pre-execution `grep -rln 'convertAgent(' tests/` confirmed that `tests/bridges/agents/convert.test.ts` is the ONLY test file that calls `convertAgent` directly. No other test files needed `mapModel: true` updates. Other test files that touch the bridge go through `prepareStagePluginAgents` (which accepts `mapModel` as an OPTIONAL field, defaulting to false); those tests do not assert on `model:` field emission, so no updates were needed.
