## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Enabled refs -> marketplace source planner | Plugin refs identify marketplace names that may or may not have source metadata. |
| Marketplace planner -> Phase 11 mutation orchestrator | Source strings become future marketplace-add inputs. |
| User/project scope plans | Same logical ref may intentionally produce independent actions in separate Pi scopes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-07 | Spoofing | official marketplace mapping | mitigate | Hard-code only `claude-plugins-official` to `anthropics/claude-plugins-official`; all other names need supported extra-known source data. |
| T-10-08 | Tampering | unsupported marketplace source shapes | mitigate | Accept only `directory` and `github.repo`; warn/skip unsupported shapes rather than broad-parsing arbitrary values. |
| T-10-09 | Information Disclosure / Tampering | Phase 10 side effects | mitigate | Architecture/source checks keep Phase 10 free of network, state mutation, notify, install, and add calls. |
| T-10-10 | Repudiation | cross-scope import actions | mitigate | Preserve `scope` on every planned marketplace/plugin/skipped/diagnostic entry. |
</threat_model>
