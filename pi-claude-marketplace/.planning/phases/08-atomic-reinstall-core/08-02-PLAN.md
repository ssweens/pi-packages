<objective>
Add reinstall-safe replacement helpers for the skills and commands bridges.

Purpose: Existing `commitPreparedSkills` and `commitPreparedCommands` are correct for install/update restage semantics, but they can delete previous resources before later failures are known. Reinstall needs backup handles that can restore old resources after bridge or state-save failures.
Output: Opaque skills/commands replacement handle types, replace/rollback/finalize helpers, and bridge unit tests.
</objective>

Existing bridge contracts to preserve:
```typescript
export async function prepareStageSkills(input: StageSkillsInput): Promise<PreparedSkillsStaging>;
export async function commitPreparedSkills(prepared: PreparedSkillsStaging): Promise<string | undefined>;
export async function abortPreparedSkills(prepared: PreparedSkillsStaging): Promise<void>;

export async function prepareStageCommands(input: StageCommandsInput): Promise<PreparedCommandsStaging>;

export async function abortPreparedCommands(prepared: PreparedCommandsStaging): Promise<void>;
```

New skills contract:
```typescript
export interface SkillsReplacement {
  readonly kind: "noop" | "replaced";
  readonly prepared: PreparedSkillsStaging;
}
export async function replacePreparedSkills(prepared: PreparedSkillsStaging): Promise<SkillsReplacement>;
export async function rollbackSkillsReplacement(replacement: SkillsReplacement): Promise<readonly string[]>;
export async function finalizeSkillsReplacement(replacement: SkillsReplacement): Promise<readonly string[]>;
```

New commands contract:
```typescript
export interface CommandsReplacement {
  readonly kind: "noop" | "replaced";
  readonly prepared: PreparedCommandsStaging;
}
export async function replacePreparedCommands(prepared: PreparedCommandsStaging): Promise<CommandsReplacement>;
export async function rollbackCommandsReplacement(replacement: CommandsReplacement): Promise<readonly string[]>;
export async function finalizeCommandsReplacement(replacement: CommandsReplacement): Promise<readonly string[]>;
```

The exact internal fields on `SkillsReplacement` and `CommandsReplacement` are bridge discretion, but the public handle must be opaque to orchestrators.

  <behavior>
    - Test 1: replacing an old skill directory with new staged content updates the target, then `rollbackSkillsReplacement` restores the old `SKILL.md` bytes.
    - Test 2: after successful replacement, `finalizeSkillsReplacement` removes backup directories and leaves staged target content in place.
    - Test 3: if a staged rename fails because an unrelated non-previous target already exists, the helper restores all backed-up previous skill dirs before throwing.
    - Test 4: noop prepared values produce noop replacement handles and rollback/finalize return an empty leak list.
  </behavior>
  <action>
    Implement reinstall-safe skills bridge helpers per D-06 and PRL-09/PRL-10.

    Concrete implementation requirements:
    - Add public type exports in `skills/types.ts` for `SkillsReplacement` or equivalent. The type must be safe for orchestrators to hold without reading `_previousNames` or `_renamePairs`.
    - In `skills/stage.ts`, add `replacePreparedSkills(prepared)`.
      - If `prepared.kind === "noop"`, return a noop replacement handle.
      - For staged values, create a backup root under `prepared.locations.skillsStagingDir` using `randomUUID()`, for example `<skills-staging>/backup-<uuid>/`.
      - Backup only previous target dirs named by `prepared._previousNames`. Do not move unrelated/manual target dirs that are not listed in `_previousNames`.
      - For each previous name, run `assertSafeName(name, "previous skill name")`, derive `target = path.join(prepared.locations.skillsTargetDir, name)`, assert containment, and if it exists rename it into the backup root. ENOENT means there was nothing to backup.
      - After backups, `mkdir(prepared.locations.skillsTargetDir, { recursive: true })`, then rename every staged pair in `prepared._renamePairs` into place.
      - If any backup/rename step fails after one or more backups or staged renames, attempt to restore via the same internal rollback routine before throwing. If rollback restoration also fails, throw an Error whose message includes `MANUAL RECOVERY REQUIRED:` and the failed path details per D-07.
    - Add `rollbackSkillsReplacement(replacement)`.
      - Remove any newly-renamed target dirs from `prepared._renamePairs` where present.
      - Restore all backup dirs to their original target paths.
      - Return a readonly string array of leak/manual-cleanup details; do not silently swallow failed restore paths.
    - Add `finalizeSkillsReplacement(replacement)`.
      - Remove the backup root with `rm(..., { recursive: true, force: true })` and return any cleanup leak detail as a string array.
    - Export the new functions and types from `skills/index.ts`.
    - Do not change `commitPreparedSkills`; existing install/update behavior must remain unchanged.
  </action>
