  <action>
    Implement reinstall-safe agents bridge helpers per D-06 and D-11 through D-14.

    Concrete implementation requirements:
    - Add public type exports in `agents/types.ts` for `ReplacePreparedAgentsOptions`, `AgentsReplacement`, or equivalent.
    - In `agents/stage.ts`, add `replacePreparedAgents(prepared, options)`.
      - If `prepared.kind === "noop"`, return a noop replacement handle.
      - If `prepared.result.failed.length > 0` and `options?.force !== true`, throw an Error whose message includes each failed `generatedName`, `targetPath`, and `reason`. This must happen before moving any file or writing `agents-index.json`.
      - If `options?.force === true`, include both `prepared._previousEntries` and `prepared._foreignPreservedEntries` in the set of target-plugin previous files to backup/replace. This is limited to this plugin's own previous agents-index rows because `prepareStagePluginAgents` already rejects cross-owner conflicts in `findOwnershipConflicts`.
      - Backup old target files into a backup root under `prepared.locations.agentsStagingDir`, for example `<agents-staging>/backup-<uuid>/`.
      - Backup the old agents index by preserving exact previous index content. Acceptable implementations: rename/copy `prepared.locations.agentsIndexPath` into the backup root when it exists, or read the exact prior index object and restore it with `saveAgentsIndex`. Missing index must restore to missing-or-empty behavior without throwing.
      - Remove backed-up previous target files, create `prepared.locations.agentsDir`, rename all `prepared._stagedFilePaths` into place, then save the new agents index.
      - New index contents after replacement must be `prepared._otherEntries + prepared._newEntries` by default; in non-force mode there are no foreign failures. In force mode, do not preserve `prepared._foreignPreservedEntries` because their target content was deliberately overwritten.
      - If any replacement step fails after mutation begins, attempt rollback before throwing. If rollback also fails, throw an Error whose message includes `MANUAL RECOVERY REQUIRED:` and failed file/index path details per D-07.
    - Add `rollbackAgentsReplacement(replacement)`.
      - Remove newly-renamed agent target files where present.
      - Restore backed-up previous agent files.
      - Restore old agents-index content or absence/empty state.
      - Return a readonly string array of leak/manual-cleanup details.
    - Add `finalizeAgentsReplacement(replacement)`.
      - Remove the backup root and return any cleanup leak detail as a string array.
    - Export the new functions and types from `agents/index.ts`.
    - Do not change `commitPreparedAgents`; existing install/update behavior must remain unchanged.
