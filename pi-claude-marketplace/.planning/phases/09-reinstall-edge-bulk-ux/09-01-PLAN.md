    Concrete TypeScript contracts to add:
    ```typescript
    export type ReinstallPluginsTarget =
      | { readonly kind: "all" }
      | { readonly kind: "marketplace"; readonly marketplace: string }
      | { readonly kind: "plugin"; readonly plugin: string; readonly marketplace: string };

    export interface ReinstallPluginsOptions {
      readonly ctx: ExtensionContext;
      readonly pi: ExtensionAPI;
      readonly scope?: Scope;
      readonly cwd: string;
      readonly target: ReinstallPluginsTarget;
      readonly force?: boolean;
    }

    export async function reinstallPlugins(
      opts: ReinstallPluginsOptions,
    ): Promise<readonly ReinstallPluginOutcome[]>;
    ```

    Concrete enumeration behavior:
    - Add private `interface ResolvedReinstallTarget { readonly plugin: string; readonly marketplace: string; readonly scope: Scope; }`.
    - Add private `async function enumerateReinstallTargets(opts: ReinstallPluginsOptions): Promise<readonly ResolvedReinstallTarget[]>`.
    - For `target.kind === "all"`, enumerate installed plugins from scopes `opts.scope === undefined ? ["user", "project"] : [opts.scope]` in that exact scope order. Load each scope with `loadState(locationsFor(scope, opts.cwd).extensionRoot)`. For every `state.marketplaces[mpName].plugins[pluginName]`, push `{ plugin: pluginName, marketplace: mpName, scope }`.
    - For `target.kind === "marketplace"` and `target.kind === "plugin"` with no explicit scope, call `resolveScopeFromState(target.marketplace, locationsFor("user", cwd), locationsFor("project", cwd))` exactly as update does.
    - For marketplace/plugin targets with explicit scope, use `{ scope: opts.scope, locations: locationsFor(opts.scope, cwd) }` directly.
    - After resolving the scope for marketplace/plugin targets, load that state. If the marketplace is absent, throw `new Error(`Marketplace "${mpName}" not found in ${resolved.scope} scope.`)`.
    - For marketplace targets, enumerate only `Object.keys(mp.plugins)`.
    - For plugin targets, return a one-item array even if the plugin is not installed; the single-plugin core will return `partition: "skipped"` with `notes: ["not installed"]`.
    - Sort the final target list with `user` before `project`, then `marketplace.localeCompare`, then `plugin.localeCompare` before executing anything.

    Concrete bulk execution behavior:
    - In `reinstallPlugins`, catch target-enumeration errors and call `notifyError(ctx, formatErrorWithCauses(err), err)`, then return `[]`.
    - If enumeration returns `[]`, call `notifySuccess(ctx, "No plugins installed.")` and return `[]`.
    - Loop targets sequentially with `for (const target of targets)`.
    - For each target call `await reinstallPlugin({ ctx, pi, scope: target.scope, cwd, marketplace: target.marketplace, plugin: target.plugin, render: "none", ...(opts.force === undefined ? {} : { force: opts.force }) })`.
    - If a per-target call unexpectedly throws, convert it to a failed outcome with `partition: "failed"`, `name`, `marketplace`, `scope`, and `notes: [formatErrorWithCauses(err)]`; continue to the next target.
    - Export `reinstallPlugins`, `ReinstallPluginsOptions`, and `ReinstallPluginsTarget` from `orchestrators/plugin/index.ts` and `orchestrators/index.ts`.
    - Do not import `DEFAULT_GIT_OPS`, `GitOps`, `refreshGitHubClone`, or `../../platform/git.ts` in `reinstall.ts`.

    Concrete rendering helpers to add:
    - `function renderReinstallPartitionAndNotify(ctx: ExtensionContext, pi: ExtensionAPI, outcomes: readonly ReinstallPluginOutcome[]): void`.
    - `function partitionReinstallOutcomes(outcomes: readonly ReinstallPluginOutcome[]): Record<ReinstallPluginPartition, ReinstallPluginOutcome[]>`.
    - `function renderReinstallPartition(lines: string[], label: "Reinstalled" | "Skipped" | "Failed", outcomes: readonly ReinstallPluginOutcome[]): void`.
    - `function formatReinstallOutcomeLine(outcome: ReinstallPluginOutcome): string`.

    Concrete output shape:
    - Partition order must be exactly `Reinstalled:`, then `Skipped:`, then `Failed:`.
    - Each non-empty partition item must render as `  - [${scope}] ${name}@${marketplace}`.
    - If `outcome.notes` has entries, append `: ${outcome.notes.join("; ")}` to that item line.
    - Empty partitions must be omitted.
    - The first line must be:
      - `Reinstalled plugin "${name}".` when exactly one outcome has `partition === "reinstalled"`.
      - `Reinstalled ${count} plugins.` when more than one outcome has `partition === "reinstalled"`.
      - `Plugin reinstall complete.` when zero outcomes were reinstalled but at least one skipped or failed outcome exists.
    - Build reload-hint inputs only from outcomes where `partition === "reinstalled" && resourcesChanged === true`. Sort those outcomes with `compareReinstallOutcome`, then pass their `name` values to `reloadHint("refresh", names)`.
    - Append soft-dependency warnings only from outcomes where `partition === "reinstalled"`: `subagentWarningIfNeeded(pi, reinstalled.flatMap((o) => o.stagedAgents ?? []))` and `mcpAdapterWarningIfNeeded(pi, reinstalled.flatMap((o) => o.stagedMcpServers ?? []))`.
    - Compose body as `summary + partitions + soft-dependency warnings`, then call `notifySuccess(ctx, appendReloadHint(body, hint))` exactly once.
    - Do not call `notifyError` for per-plugin failed outcomes from the batch renderer; the `Failed:` partition is the deterministic user-visible error report for batch mode.
